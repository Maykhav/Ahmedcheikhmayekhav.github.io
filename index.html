              01
              IoT Industriel · Modbus RTU · BDD
              Stage Industriel
            

            
Système d’Acquisition Industrielle

            

              Contexte : Chez Wisdom Design Technologies (Tunis, 2024), les lignes de
              production industrielles manquaient d’un système de collecte de données centralisé,
              fiable et horodaté pour le suivi et l’analyse de production.
            

            

              Solution : Conception et implémentation d’un système complet d’acquisition
              avec communication Modbus RTU (RS-232/RS-485), intégration RTC pour l’horodatage précis
              des mesures, et enregistrement automatique dans une base de données industrielle.
            

            
    
              
  • Communication Modbus RTU sur RS-232/RS-485 avec les équipements terrain
    • 
              
  • Horodatage précis via RTC — traçabilité complète des mesures
    • 
              
  • Enregistrement automatique en base de données (logging continu)
    • 
              
  • Architecture IoT industrielle scalable pour plusieurs lignes de production
    • 
            

            

              Modbus RTURS-232/RS-485RTCPythonBase de donnéesIoT industriel
            

          

          

            

              
            

          

        


        
        

          

            

              02
              Python · MQTT · Dashboard · Cloud
            

            
Système de Supervision IoT Connecté

            

              Problème : Les données de capteurs restent silotées dans les équipements —
              sans agrégation, sans visualisation, sans alertes. L’opérateur est aveugle.
            

            

              Solution : Pipeline MQTT complet : collecte des données capteurs,
              persistance en SQLite avec horodatage, synchronisation vers base de données cloud,
              et visualisation en temps réel via dashboard web.
            

            
    
              
  • Broker MQTT pour la collecte multi-capteurs en temps réel
    • 
              
  • Base de données SQLite locale + synchronisation cloud
    • 
              
  • Dashboard web avec visualisation graphique en direct
    • 
              
  • Architecture extensible : ajout de capteurs sans modifier le backend
    • 
            

            

              PythonMQTTSQLiteCloud BDDDashboard Web
